# Анализ видео столика с детекцией человека

Проект позволяет анализировать видеозаписи столика и определять моменты, когда человек подходит к столу или покидает его. Используются два метода детекции: **YOLOv8** (нейросеть) и **OpenCV** (вычитание фона).  

---

```bash
git clone https://github.com/maxexplorer/PythonCV

```
cd dodopizza_test_task

```
pip install -r requirements.txt
```

Примечание:

Требуется Python >=3.10.

Запуск
1. Через графический интерфейс (UI)
```
python main.py
```

После запуска открывается окно, где можно:

- Выбрать видеофайл (.mp4, .avi, .mov).
- Выбрать детектор: YOLO или OpenCV.
- Включить или отключить запись выходного видео.
- Нажать "Запустить" для анализа. 
2. Через командную строку (CLI)

```
python main.py --video video1.mp4
```

В этом режиме:

- Используется детектор YOLO.
- Опция записи видео отключена.
- ROI выбирается автоматически на первом кадре (как при UI).
- Метрики выводятся в консоль.

## Логика детекции
YOLODetector
- Использует YOLOv8 для обнаружения объектов person в выбранной ROI.
- Если хотя бы один человек пересекает ROI, событие фиксируется.
OpenCVDetector
- Использует вычитание фона (BackgroundSubtractorMOG2) в ROI.
- Контуры с площадью >500 пикселей считаются наличием человека.

## FSM — отслеживание событий
Состояния столика:
- Empty — стол пуст.
- Full — человек за столом.

События:
- Empty_to_Full — стол стал занятым.
- Full_to_Empty — стол опустел.

Каждое изменение состояния сохраняется с временной меткой.

Среднее время задержки (Mean idle time) вычисляется как средняя длительность, когда стол оставался пустым между Full_to_Empty и Empty_to_Full.
Результат
При запуске выводится таблица событий и среднее время задержки. Пример:

    timestamp          event
0        0.00  Empty_to_Full
1        0.04  Full_to_Empty
2        0.20  Empty_to_Full
3       17.36  Full_to_Empty
4       18.92  Full_to_Empty
Mean idle time: 1.15 sec

- timestamp — время события в секундах.
- event — событие перехода состояния.
- Mean idle time — среднее время, когда стол был пуст.
Примечания
- ROI выбирается на первом кадре видео.
- Для больших видео рекомендуется включить запись видео только при необходимости.
- Пример проблемного кадра можно сохранить через OpenCV при отладке.
